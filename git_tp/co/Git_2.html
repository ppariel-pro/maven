Ajouter un fichier

Commençons par l'ajout d'un fichier.

Par exemple, créons un fichier myfile.txt avec un petit texte dedans.

1
echo "Ceci est un test de git" > myfile.txt 
echo "Ceci est un test de git" > myfile.txt

Voyons la façon dont git perçoit ce nouveau fichier

1
git status
git status

Il nous indique qu'on est sur la branche master, qu'il y a un fichier myfile.txt mais qu'il n'est pas suivi (« untracked ») par git.

Comme nous voulons intégrer ce fichier au projet, on ne peut pas encore faire le commit ,car commit n'envoie que les fichiers qui sont *tracked*, c'est à dire dans l'index (*staging*). Ajoutons le fichier, comme git nous le suggère, avec add

1
git add myfile.txt
git add myfile.txt

On refait un

1
git status
git status

Et, cette fois, git nous répond

Le fichier myfile.txt sera bien intégré dans notre prochain commit

1
git commit -m "mon premier commit"
git commit -m "mon premier commit"

Remarquons ici qu'avec -m, nous avons choisi de préciser le message de commit directement sur la ligne de commande.

En lançant git commit tout court, l'éditeur de texte ($EDITOR) s'ouvre automatiquement pour inviter à saisir un commentaire de soumission.

1
[master (root-commit) 2474c83] mon premier commit
2
1 file changed, 1 insertion(+)
3
create mode 100644 myfile.txt
